## 1. Data Ownership Specifications

The following matrix defines the authoritative owner of each domain property:

| Aggregate Root       | Concept / Property    | Authoritative Owner     | Purpose                                    |
| :------------------- | :-------------------- | :---------------------- | :----------------------------------------- |
| **Job Aggregate**    | Payload Data          | `Job`                   | JSON payload of execution task parameters. |
|                      | Priority Level        | `Job`                   | priority value used in claims sorting.     |
|                      | Scheduling Metadata   | `Job`                   | `scheduled_at` timestamp.                  |
|                      | Execution Status      | `Job`                   | Active state of the job.                   |
|                      | Exit Logs / Output    | `JobExecution`          | stdout logs generated by the worker.       |
|                      | Run Duration          | `JobExecution`          | time elapsed during processing.            |
|                      | Current Attempt Index | `JobExecution`          | number of execution runs completed.        |
| **Worker Aggregate** | Capabilities          | `Worker`                | runtime version details, parallel limits.  |
|                      | Registration State    | `Worker`                | active/idle/offline status.                |
|                      | Last Heartbeat Check  | `Worker`                | health check timestamp.                    |
|                      | Expiration Lock       | `Lease`                 | lease key TTL in Redis.                    |
|                      | Owner Identifier      | `Lease`                 | active worker UUID association.            |
| **Queue Aggregate**  | Max Concurrency Cap   | `Queue`                 | queue concurrency limits.                  |
|                      | Retry Configuration   | `Queue` / `RetryPolicy` | retry rules (max attempts, backoffs).      |
|                      | Schedule Settings     | `Queue`                 | cron configuration rules.                  |
